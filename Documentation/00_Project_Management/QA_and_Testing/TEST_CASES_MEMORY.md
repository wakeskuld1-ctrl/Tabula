# 测试用例记忆库

## 目标
记录项目内已发现问题、潜在风险与对应测试用例，形成长期可复用的测试资产。

## 覆盖范围
- 粘贴与批量更新流程
- 列插入与类型推断
- 聚合公式与后端执行
- 前端列类型标记与渲染

## 已知问题与回归用例

### 问题 1：粘贴后多写入一段空数据

**可能问题点**
- 粘贴数据尾部空行/空列未裁剪导致扩容
- 后端批量更新对全空更新仍触发扩行

**用例**
1. 场景：粘贴包含尾部空行  
   - 步骤：复制 2 行有效数据 + 1 行空行，粘贴到表格  
   - 期望：仅写入 2 行，不新增空行  
2. 场景：粘贴包含尾部空列  
   - 步骤：复制 2 列有效数据 + 1 列空列，粘贴到表格  
   - 期望：仅写入 2 列，不新增空列  
3. 场景：粘贴中间夹空行  
   - 步骤：复制 3 行数据，其中第 2 行为空，粘贴到表格  
   - 期望：中间空行保留，尾部不扩容  

### 问题 2：SUM(Utf8) 聚合报错

**可能问题点**
- 新列默认 Utf8 类型，数值列未被正确推断
- 聚合公式对字符串列执行 SUM

**用例**
1. 场景：插入新列后粘贴数值并执行 SUM  
   - 步骤：插入新列，粘贴 10/20/30，输入公式 SUM(该列)  
   - 期望：返回 60，无类型错误  
2. 场景：混合数值与空值  
   - 步骤：插入新列，粘贴 10/空/30，执行 SUM  
   - 期望：返回 40，无类型错误  
3. 场景：混合数值与非数值  
   - 步骤：插入新列，粘贴 10/ABC/30，执行 SUM  
   - 期望：类型推断回落为文本，提示不可聚合或返回可预测错误  

### 问题 3：execute_sql 与 get_plan 链路回归

**可能问题点**
- 自愈修复路径未触发或触发后返回结构变化
- 逻辑/物理计划失败时错误提示不稳定
- 空结果集返回结构不一致

**用例**
1. 场景：缺失字段自愈修复（execute_sql）  
   - 步骤：执行包含未引号字段名的 SQL，触发 "No field named" 后重试  
   - 期望：最终返回正常结果或明确错误，日志包含修复与重试记录  
2. 场景：空结果集（execute_sql）  
   - 步骤：查询空表或无匹配结果的 SQL  
   - 期望：columns/rows 为空，error 为空  
3. 场景：逻辑计划失败（get_plan）  
   - 步骤：提交语法错误的 SQL  
   - 期望：physical_plan_text 以 "Logical Plan Error" 开头，warnings 包含错误  
4. 场景：物理计划失败（get_plan）  
   - 步骤：提交能通过语法但物理计划构建失败的 SQL  
   - 期望：physical_plan_text 以 "Physical Plan Error" 开头，warnings 包含错误  
5. 场景：正常计划输出（get_plan）  
   - 步骤：提交合法 SQL  
   - 期望：plan_json 与 physical_plan_text 非空，estimated_rows/estimated_bytes 合理

### 问题 4：上传链路回归

**可能问题点**
- 多 Sheet 文件返回结构不稳定
- 目录不可写或文件写入失败时错误信息不一致
- 不支持类型未明确标识
- 表名清洗规则与注册表名不一致

**用例**
1. 场景：多 Sheet Excel 触发选择  
   - 步骤：上传包含多个 sheet 的 xlsx  
   - 期望：status=select_sheet，返回 sheets 与 sanitized_name  
2. 场景：目录不可写  
   - 步骤：模拟 data 目录不可写后上传文件  
   - 期望：status=error，message 提示 Failed to create data dir  
3. 场景：文件写入失败  
   - 步骤：模拟写入失败（磁盘满或权限）后上传  
   - 期望：status=error，message 提示 Failed to write file  
4. 场景：不支持文件类型  
   - 步骤：上传不支持扩展名文件  
   - 期望：status=ok，message 包含 not registered, unsupported type  
5. 场景：表名清洗  
   - 步骤：上传包含空格/括号/点的文件名  
   - 期望：返回 table 字段为清洗后的表名

### 问题 5：register_table 链路回归

**可能问题点**
- 表名清洗与数据源注册不一致
- 转码失败回退路径与消息不稳定
- sqlite 内部表名与外部表名绑定错误
- metadata 持久化缺失或字段不一致

**用例**
1. 场景：表名清洗  
   - 步骤：register_table 提交 datafusion.public.ABC123  
   - 期望：注册表名为 ABC123，日志包含清洗提示  
2. 场景：Excel 转码失败回退  
   - 步骤：制造转码失败后调用 register_table  
   - 期望：仍可注册成功且回退为原始 Excel 源  
3. 场景：SQLite 内部表名回退  
   - 步骤：source_type=sqlite 且 sheet_name 为空  
   - 期望：内部表名使用清洗后的表名  
4. 场景：metadata 持久化  
   - 步骤：成功注册后查询 metadata 表  
   - 期望：file_path/source_type/sheet_name/header_rows/header_mode 与输入一致

## 类型推断与列类型标记

**可能问题点**
- 类型推断算法对空值、混合值边界处理不一致
- 前端列类型标记与后端 schema 不一致

**用例**
1. 场景：纯整数列  
   - 步骤：粘贴 1/2/3 到新列  
   - 期望：列类型为 Int64，前端列头显示数值图标  
2. 场景：浮点列  
   - 步骤：粘贴 1.2/3.4 到新列  
   - 期望：列类型为 Float64，前端列头显示数值图标  
3. 场景：混合整数与小数  
   - 步骤：粘贴 1/2.5 到新列  
   - 期望：列类型为 Float64  
4. 场景：混合数值与文本  
   - 步骤：粘贴 1/ABC 到新列  
   - 期望：列类型为 Utf8，前端列头显示文本图标  
5. 场景：全为空值  
   - 步骤：粘贴空值到新列  
   - 期望：列类型为 Utf8  

## 批量更新与扩容

**可能问题点**
- 只包含空值的更新触发扩行
- 更新范围跨分页边界

**用例**
1. 场景：仅空值更新  
   - 步骤：批量更新一行为空值  
   - 期望：不新增行  
2. 场景：跨分页更新  
   - 步骤：在分页边界附近粘贴多行数据  
   - 期望：所有行正确写入且不重复

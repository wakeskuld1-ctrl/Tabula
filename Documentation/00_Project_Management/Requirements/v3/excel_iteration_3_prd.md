# Excel 类表格能力建设 - 第三轮迭代需求文档 (PRD)

**版本:** v1.0  
**状态:** Draft  
**日期:** 2026-02-04  

---

## 1. 概述 (Overview)

本轮迭代主要补齐 Excel 表格的高频核心交互能力，重点解决大数据量下的数据查看与筛选诉求。
核心包含三大特性：**行级选择 (Row Selection)**、**行列冻结 (Freeze Panes)** 以及 **后端驱动的表头筛选 (Backend-driven Header Filter)**。

---

## 2. 详细需求 (Detailed Requirements)

### 2.1 行选择 (Row Selection)

#### 2.1.1 需求描述
用户需要能够像在 Excel 中一样，通过点击行号区域来选中整行数据，以便进行后续的行级操作（如删除行、复制行等，虽本期可能仅做选中视觉）。

#### 2.1.2 用户故事 (User Stories)
*   **US-1:** 作为用户，我点击 Grid 左侧的行号数字时，该行的所有单元格应变为选中状态。
*   **US-2:** 作为用户，我按住 `Shift` 点击另一行号时，应选中这两行之间的所有行。
*   **US-3:** 作为用户，我按住 `Ctrl` (Mac下为Cmd) 点击不连续的行号时，应追加选中这些不连续的行。

#### 2.1.3 交互与冲突处理逻辑
*   **区域划分:**
    *   **列头区 (A, B, C...):** 点击触发列选中。
    *   **行头区 (1, 2, 3...):** 点击触发行选中。
    *   **筛选图标区:** 优先触发筛选菜单，不触发选择。
*   **冲突原则:** 遵循 Excel 原生逻辑。行选与列选互不干扰操作触发，但视觉上通常是“最后一次操作”决定当前的 Selection 状态。

### 2.2 行列冻结 (Freeze Panes)

#### 2.2.1 需求描述
在查看宽表或长表时，用户需要固定顶部表头或左侧关键列（如ID、名称），以便在滚动时始终保持上下文可见。支持任意位置的冻结，而非仅限首行首列。

#### 2.2.2 用户故事 (User Stories)
*   **US-4:** 作为用户，我选中任意单元格（例如 C5），选择“冻结窗格”功能后，该单元格左侧的列（A, B）和上方的行（1-4）应被固定。
*   **US-5:** 冻结后，当我不停向右滚动时，A、B 列始终停留在屏幕左侧。
*   **US-6:** 冻结后，当我不停向下滚动时，第 1-4 行始终停留在屏幕顶部。

#### 2.2.3 技术规格
*   **参数:** 前端 Grid 组件需接收 `freezeColumns` (int) 和 `freezeRows` (int) 属性。
*   **限制:** 冻结区域不能超过视口可视范围的 80%，否则应提示或限制，防止数据区不可见（参考 Excel 行为，或 GlideGrid 自身限制）。

### 2.3 表头筛选 (Header Filter) - 重点

#### 2.3.1 需求描述
支持对任意列进行值筛选。由于数据量可能达到百万级，前端无法全量加载所有唯一值，因此必须采用 **后端驱动 + 懒加载** 模式。

#### 2.3.2 业务逻辑与流程

1.  **入口 (Entry):**
    *   列头右侧显示“漏斗”图标。
    *   点击图标展开筛选下拉菜单。

2.  **唯一值获取 (Unique Values - Lazy Load):**
    *   **触发时机:** 仅在菜单展开时请求后端。
    *   **查询逻辑:** 向后端发送请求，获取该列的唯一值列表 (`DISTINCT`)。
    *   **级联筛选 (Cascading):**
        *   查询 B 列的唯一值时，**必须**带上当前 A 列、C 列等已生效的过滤条件。
        *   *例子:* 如果 A 列已筛选 "Region = Asia"，则点开 B 列 "Country" 时，只显示属于 Asia 的国家。
        *   *后端实现:* `SELECT DISTINCT column_b FROM table WHERE (Current Global Filters) LIMIT 20`。
    *   **数量限制:** 为保证性能，筛选列表仅动态展示前 **20** 个（或 100 个）唯一值。支持搜索框进行模糊搜索以查找更多。

3.  **应用筛选 (Apply):**
    *   用户勾选若干值后，点击“确认”。
    *   前端生成 Filter 对象（如 `{ col: "status", operator: "in", values: ["open", "closed"] }`）。
    *   Grid 重新向后端请求数据页，后端拼接 SQL `WHERE status IN ('open', 'closed')` 返回结果。

#### 2.3.3 用户故事 (User Stories)
*   **US-7:** 点击表头筛选图标，能快速看到该列的前 20 个唯一值。
*   **US-8 (级联):** 当我先筛选了“年份=2024”，再点开“月份”筛选时，只应该看到 2024 年存在的月份，不应出现 2023 年的数据干扰。
*   **US-9:** 我可以通过搜索框输入文本，触发后端模糊查询，找到列表前 20 个之外的值。

## 3. 验收标准 (Acceptance Criteria)

1.  **行选:** 能准确选中单行、多行；行号高亮与 Excel 一致。
2.  **冻结:** 滚动时冻结区域纹丝不动；解冻后恢复正常滚动。
3.  **筛选:**
    *   筛选菜单弹出速度 < 500ms (依赖后端查询速度)。
    *   级联逻辑正确：前置条件变化后，后置列的筛选值列表随之变化。
    *   数据准确性：筛选后的 Grid 数据与数据库 `WHERE` 查询结果一致。

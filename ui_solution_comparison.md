# 前端表格方案深度对比：Glide vs Univer vs FortuneSheet

## 1. 核心定位差异

| 特性 | Glide Data Grid (当前方案) | Univer (推荐方案) | FortuneSheet / Luckysheet |
| :--- | :--- | :--- | :--- |
| **核心定位** | **高性能数据网格组件** | **全栈 Office 文档框架** | **在线电子表格应用** |
| **类似产品** | Ag-Grid, Handsontable | Google Sheets, Excel 365 | Luckysheet (原版) |
| **主要用途** | 大数据展示、分析、即席查询 | 完整的在线文档协作、复杂报表 | 传统 Excel 替代、填报 |
| **渲染技术** | Canvas (高性能) | Canvas (高性能) | Canvas (高性能) |
| **UI 复杂度** | **极简** (仅网格，无工具栏) | **完整** (Ribbon 工具栏、右键菜单) | **完整** (类似 Excel 2013) |
| **数据量级** | **百万级** (虚拟滚动优秀) | **千万级** (官方宣称) | 十万级 (纯前端易卡顿) |
| **定制难度** | 高 (需手写 UI) | 中 (插件化架构) | 低 (开箱即用) |

## 2. 为什么当前看起来 "比较简陋"?
**Glide Data Grid** 本质上是一个**渲染引擎**。它提供了极致的滚动性能（60FPS 渲染百万行），但它**不包含**以下“成熟 Excel”的功能：
*   ❌ 顶部工具栏 (加粗、颜色、字体)
*   ❌ 公式编辑栏 (=SUM(A1:B1))
*   ❌ 底部 Sheet 切换条
*   ❌ 侧边属性面板
*   ❌ 复杂的右键菜单

如果您需要的是一个“看起来像 Excel”的界面，Glide 需要我们**从头开发**这些周边 UI。

## 3. 推荐方案：切换至 Univer
如果您期望的是“成熟的 Excel 方案”，**Univer** 是目前最先进的选择：
*   **架构先进**：基于 TypeScript 的插件化架构，比 FortuneSheet (Luckysheet) 更易扩展。
*   **功能完备**：自带完整的 Ribbon 工具栏、公式引擎、协同编辑能力。
*   **性能强大**：同样基于 Canvas 渲染，官方宣称支持千万级单元格。
*   **后端集成**：支持 Node.js SDK，方便与我们的 Rust 后端对接。

## 4. 建议行动路径
1.  **保留 Glide**：作为“纯数据浏览模式” (Data View)，用于快速查看超大数据集。
2.  **集成 Univer**：作为“沙盘编辑模式” (Sheet View)，提供完整的 Excel 体验（公式、格式、图表）。
3.  **前端改造**：在 `App.tsx` 中增加视图切换，允许用户在“极速网格”和“全功能表格”之间切换。

# 公式用法提示上屏（Formula Help Drawer）设计

日期：2026-03-15  
作者：Codex

## 背景与目标
- 需求：将“文档里的函数用法”直接展示在前端界面，提供可检索的提示。
- 约束：不反向解析 README，复用生成器输出，保证同源一致。
- 目标：在界面提供公式帮助抽屉（搜索 + 全量表格），双语展示。

## 方案选择
已确认采用方案A：
- 顶部按钮触发右侧抽屉面板。
- 面板内提供搜索与全量公式表格。
- 数据来源为生成器输出的结构化 JSON。

## 架构设计
1) 数据输出
   - 生成器在写 README 的同时生成 `frontend/src/data/formula_help.json`。
2) UI 组件
   - App 顶部状态栏新增“公式帮助 / Formula Help”按钮。
   - 右侧抽屉包含搜索框与公式列表（函数名/语法/示例/参数说明/用途）。
3) 筛选逻辑
   - 关键词匹配函数名、语法、用途、参数说明。

## 数据流
- 前端启动时导入 JSON 数据。
- 输入搜索词 → 过滤列表 → 即时渲染。

## 错误处理
- JSON 为空或加载失败：显示空态提示，不影响主表格。

## 测试策略
- 纯函数过滤逻辑的单元测试（vitest）。
- UI 抽屉可开关、搜索可过滤的手测验证。

## 交付物
- 设计文档：`docs/plans/2026-03-15-formula-help-ui-design.md`
- 公式提示 JSON 生成与 UI 展示
- CI 保持 README 与 JSON 一致性

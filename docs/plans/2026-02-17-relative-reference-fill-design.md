# 相对引用位移与填充序列策略设计文档

---
  
## 1. 目标与范围
- **目标**：为 Excel 下拉填充提供“相对引用位移 + 智能序列推断”的可实现设计，支撑后续实现与测试。  
- **范围**：前端网格内的公式/值填充规则与数据流设计，不涉及 UI 手柄或拖拽交互细节。  
- **非目标**：不引入新的公式解析依赖，不调整后端接口。  
  
## 2. 现状与约束
- **公式引擎现状**：当前仅使用 HyperFormula 进行计算与取值，未暴露 AST/解析接口。[FormulaEngine.ts](file:///d:/Rust/metadata/frontend/src/utils/FormulaEngine.ts)  
- **网格批量更新入口**：前端已有批量编辑通道，可用于填充落地。[GlideGrid.tsx:L1854-L1968](file:///d:/Rust/metadata/frontend/src/components/GlideGrid.tsx#L1854-L1968)  
- **约束**：公式列为只读，填充需跳过。  
  
## 3. 相对引用位移策略（方案 A）
- **核心思路**：在前端实现 A1 公式解析与相对引用位移重写，支持 $ 绝对引用。  
- **解析范围**：  
  - 识别列名 + 行号：`A1`, `AA10`  
  - 识别绝对引用：`$A$1`, `$A1`, `A$1`  
  - 识别区域引用：`A1:B5`  
  - 其他内容保持原样（函数名、字符串、数字）。  
- **位移规则**：  
  - 相对列：`A` 在 `dx` 位移时变更为 `A+dx`  
  - 相对行：`1` 在 `dy` 位移时变更为 `1+dy`  
  - 绝对列/行保持不变  
  - 位移结果若列越界或行 < 1，回退为原引用或复制首值  
  
## 4. 填充序列策略（智能填充）
- **数值序列**：  
  - 若源区间长度 ≥ 2 且呈等差序列，继续等差递增  
  - 否则默认复制首值  
- **日期序列**：  
  - 识别常见日期格式（YYYY-MM-DD、YYYY/MM/DD），按天递增  
  - 若解析失败则回退复制  
- **文本+数字**：  
  - 识别尾号数字（如 Item 1），保持前缀不变，数值递增  
  - 无尾号则复制  
- **公式**：  
  - 公式以相对引用位移重写  
  - 不支持解析的公式原样复制  
  
## 5. 数据流与落地
- **输入**：源范围值、目标范围坐标、位移向量（dx, dy）  
- **处理**：  
  1) 判定单元格类型（公式/数值/日期/文本序列）  
  2) 生成目标范围二维矩阵  
  3) 统一交由批量更新通道提交  
- **输出**：目标范围更新集合（row, col, val）  
  
## 6. 错误处理与边界策略
- **只读列**：跳过填充并提示  
- **合并单元格**：若填充范围跨越合并区域，阻止或裁剪  
- **跨页填充**：允许异步加载缺页，未加载区域先用源区间推断  
  
## 7. 测试建议
- **单元测试**：  
  - 公式位移：`A1`/`$A1`/`A$1`/`$A$1` 的偏移正确性  
  - 数值序列：等差递增、非等差回退复制  
  - 文本序列：尾号递增与无尾号复制  
- **集成测试**：  
  - 跨页填充、只读列跳过、批量更新数量一致

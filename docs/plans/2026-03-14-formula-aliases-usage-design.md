# 公式参数别名与用途说明设计

日期：2026-03-14  
作者：Codex

## 背景与目标
- 已具备全量公式表格自动生成能力，但语法仍偏技术化（number1/text1）。
- 需求补充：**参数别名/说明**更贴近业务术语，且**公式用途（中文）补全**，同时保持中英文完整。
- 同步要求：脚本接入 CI，确保 README 自动校验。

## 方案选择
已确认采用方案A：
- 类型别名 + 常见函数覆盖 + CI 校验接入。

## 架构设计
1) **别名来源**
   - **typeAliases**：按参数类型生成中英文别名（通用）。
   - **functionAliases**：为常见函数提供参数别名覆盖（更贴近业务）。
2) **用途说明**
   - **functionPurposeOverrides**：为常见函数补精确用途描述（中英双语）。
   - **purposeRules**：对其余函数按命名模式归类，生成通用用途描述（中英双语）。
3) **README 输出**
   - 表格新增“参数说明 / Parameter Notes”和“用途 / Purpose”两列。
   - 元数据缺失时：语法/示例/参数说明/用途显示“—/不可用”，备注提示“待清理”。

## 数据流
1) 读取 HyperFormula 注册函数列表（全量）。
2) 尝试读取每个函数参数元数据：
   - 若存在：生成语法 + 示例 + 参数别名 + 用途描述。
   - 若缺失：输出不可用标记，备注待清理。
3) 将表格注入两份 README 的标记块。

## 错误处理
- 缺失标记块：抛错并退出。
- 函数列表为空：输出提示行，避免空白。

## 测试策略
- node:test 覆盖：
  - 表头新增列存在。
  - 参数说明/用途列生成。
  - 注入替换逻辑正确。
- CI 校验：
  - 运行 `docs:formulas:check`，若 README 不一致则失败。

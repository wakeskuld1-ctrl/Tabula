# UI Polishing + Formula Persistence Design

**Date:** 2026-03-15

## 背景
- 底部 debug overlay 显示“Loaded ... rows”后没有自动隐藏，占用视线。
- SheetBar 的“+”按钮在最右侧，需要紧贴最后一个 Sheet 标签。
- Pivot 按钮点击反馈不符合预期（提示文案不匹配、可能因字段未就绪）。
- 公式输入后会被计算成数值，导致公式原文丢失，动态依赖失效。

## 目标
- “Loaded ... rows” 提示在加载完成后保留 10 秒自动隐藏。
- “+”按钮紧贴 Sheet 标签右侧。
- Pivot 点击能稳定打开并给出准确提示。
- 公式原文保留（不改 DAG），单元格显示计算值但公式栏仍显示公式。

## 非目标
- 不改后端 DAG/计算规则。
- 不改后端接口行为，仅修前端数据流。

## 设计

### 1) Debug Overlay 自动隐藏
- 当 debugInfo 匹配 `Loaded ... rows` 且 loading=false 时，启动 10 秒定时清空。
- 其它提示不自动清空。

### 2) Sheet “+” 位置
- 将 “+” 按钮移入 `.sheet-tabs` 容器末尾。
- 使用 CSS 让 “+” 紧贴最后一个标签。

### 3) Pivot 按钮
- 如果表数据/字段尚未加载，先触发 `fetchTableData` 再打开 Pivot。
- 修正 Pivot 应用后的错误提示文案，避免出现“切换会话”误导。

### 4) 公式原文保留（不改 DAG）
- 保存公式原文到缓存与 FormulaEngine（rawValue = `=...`）。
- 计算值用于显示：
  - 默认由 FormulaEngine 计算（DAG 依赖保持）。
  - 对后端拦截计算的公式（SUM/XLOOKUP/VLOOKUP）使用展示覆盖值。
- 后端更新时仍发送公式原文作为 new_value（保持持久化）。

## 风险
- 若后端不接受公式字符串，需与后端协调字段类型。
- 公式显示覆盖值可能需要在后续刷新时重新计算（已保持现有刷新逻辑）。

## 验收
- Loaded 提示在加载完成 10 秒后消失。
- “+” 贴在最后一个 Sheet 标签右侧。
- Pivot 点击后稳定打开，无误导性提示。
- 公式栏显示原公式，单元格显示计算值，依赖更新时可刷新。

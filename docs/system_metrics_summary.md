# 系统关键指标与能力清单

基于最新的高负载压测 (v2.0) 及架构设计，整理了以下核心指标供核对：

## 1. 性能指标 (Performance)
| 指标项 | 实测值/设定值 | 说明 |
| :--- | :--- | :--- |
| **吞吐量 (Throughput)** | **8,444 行/秒** | 50 并发下的持续处理能力 |
| **数据处理量级** | **百万级 (1,000,000 Rows)** | 单次压测处理总行数 (耗时 ~2分钟) |
| **并发能力** | **50 线程 (无阻塞)** | 16分段锁 (Sharded Lock) 彻底消除了写锁竞争 |
| **查询延迟** | **毫秒级响应** | 热点数据走 L2 内存，非热点走 L1 Parquet |

## 2. 资源限制与阈值 (Resource Limits)
| 指标项 | 阈值设定 | 触发行为 |
| :--- | :--- | :--- |
| **内存上限 (L2 Cap)** | **64 MB** (可配) | 超过即触发 Eviction Pool 驱逐 |
| **磁盘水位 (Disk Cap)** | **85% ~ 92%** | >85% 告警，>90% 触发 L1 文件清理 |
| **单批次大小** | **默认 8192 行** | RecordBatch Compaction 目标大小，优化顺序读 |

## 3. 缓存生命周期 (Lifecycle & TTI)
| 策略阶段 | 时间/条件 | 目的 |
| :--- | :--- | :--- |
| **试用期 (Probation)** | **30 秒** | 访问 < 2 次的数据，快速淘汰防止污染 |
| **保护期 (Protected)** | **5 分钟** | 访问 ≥ 2 次的数据，常驻内存服务热点 |
| **淘汰扫描间隔** | **10 秒** | 周期性清理过期数据 |

## 4. 稳定性与容错 (Stability)
| 机制 | 表现 | 验证结果 |
| :--- | :--- | :--- |
| **防击穿 (Singleflight)** | **100% 请求合并** | 高并发下同一 Key 仅执行一次 IO |
| **内存防爆 (OOM)** | **稳定在水位线** | Log-Linear 评分精准剔除低价值数据 |
| **磁盘满载保护** | **优雅降级** | 写入失败时不崩贵，仅降级为无缓存模式 |

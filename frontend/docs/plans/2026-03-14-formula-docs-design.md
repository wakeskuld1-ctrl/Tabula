# 公式全量用法文档（README 折叠块）设计

日期：2026-03-14  
作者：Codex

## 背景与目标
- 需求：README 内必须覆盖 **HyperFormula 注册的全部函数**，并给出用法（语法 + 示例）。
- 约束：公式数量可能很大，需避免 README 过于冗长影响阅读体验。
- 目标：在 README 内提供全量、可追溯、可扩展的公式用法清单，且尽量自动化生成。

## 方案选择
- 已选：**方案C**  
  - 将全量函数表格放在 `<details>` 折叠块中，README 主体仍保持简洁。

## 架构设计
1) **函数来源**  
   - 使用 `HyperFormula.getRegisteredFunctionNames()` 获取全量函数名。
2) **语法/示例生成**  
   - 若可从 HyperFormula 元数据读取参数签名，生成类似 `FUNC(text, number, [optional])` 的语法。  
   - 若元数据不可用，语法降级为 `FUNC(...)`。  
   - 示例生成规则（模板级）：  
     - 文本参数 → `"text"`  
     - 数值参数 → `1`  
     - 布尔参数 → `TRUE`  
     - 范围参数 → `A1:A5`  
     - 未知参数 → `...`
3) **README 展示**  
   - 新增“公式样例（全量）”章节。  
   - `<details>` 内表格列：函数名 / 语法 / 示例 / 备注。

## 数据流设计
1) 读取函数列表（全量）。  
2) 逐个函数生成语法字符串与示例字符串。  
3) 按函数名排序，保证输出稳定、便于 diff。  
4) 生成 Markdown 表格写入 README 的 `<details>` 块内。

## 错误处理
- 若无法读取参数元数据：  
  - 语法使用 `FUNC(...)`  
  - 示例使用 `=FUNC(...)`  
  - 备注标明“参数元数据不可用，示例为占位”  
- 若函数列表为空：  
  - 表格输出提示行，避免 README 空白。

## 测试策略
### 自动化检查（轻量）
- 断言 README 生成内容包含表头。  
- 断言表格行数与函数数目一致。  

### 手工检查
- README `<details>` 块展开可读。  
- 示例格式正确且无明显排版错乱。

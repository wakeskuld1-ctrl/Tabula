# Fill Handle + Formula Shift (Parser) 设计文档

## 背景与目标
- 目标 1：补齐 Excel 风格的填充把手（拖拽 + 双击）能力。
- 目标 2：双击填充时，按“相邻列连续数据范围”自动扩展填充目标。
- 目标 3：公式填充时支持相对/绝对引用，并扩展到整列/整行引用位移。
- 目标 4：前端计算填充结果后批量写入后端，后端不新增接口与逻辑。

## 非目标（本次不做）
- 不实现多用户协同与单元格锁。
- 不在后端做公式解析/校验（仅在文档中记录未来升级）。
- 不引入完整公式求值引擎（仅解析与引用位移）。

## 现状摘要
- 已存在 `fillRange`：可推断序列、支持公式引用位移（仅 A1 + 行号）。
- `fillRange` 最终走 `onCellsEdited` -> `batchUpdateCells`，后端只接收最终值。
- 当前 `shiftFormulaReferences` 仅支持带行号的 A1 引用，无法处理 `F:F`/`3:3`。

## 方案选型
### 解析库
- 采用 `excel-formula-parser`（解析-only）。
  - 原因：只需引用位移，无需前端求值；解析-only 更轻量、风险更低。
  - 产出：以 AST/token 为基础进行引用改写，再重建公式文本。

## 设计方案
### 1) 交互与触发
- 拖拽填充：通过 DataEditor 的 `fillHandle` 与 `onFillPattern`。
- 双击填充：
  - 依据当前选区，找到相邻列连续有值的范围。
  - 优先使用左侧列；左侧为空时使用右侧列。
  - 仅向下扩展，避免向上或左右误填。

### 2) 公式位移（解析-only）
- 对 `=` 开头的单元格文本进行解析与位移。
- 支持引用类型：
  - A1 引用（相对/绝对）：`A1` / `$A$1` / `A$1` / `$A1`
  - 整列引用：`F:F` / `$F:$F`
  - 整行引用：`3:3` / `$3:$3`
- 位移规则：
  - 相对引用按 dx/dy 位移；绝对引用保留。
  - 不改变工作表/外部引用前缀（仅改当前引用部分）。
- 容错策略：
  - 解析失败则回退到现有 `shiftFormulaReferences`（保证可用性）。

### 3) 数据流与后端交互
- `onFillPattern`/双击计算目标范围 -> 调用 `fillRange`
- `fillRange` 生成最终值 -> `onCellsEdited` -> `batchUpdateCells`
- 后端只接收最终值 `{ row, col, val }`，不感知“填充规则”。

### 4) 性能与稳定性
- 仅对填充范围内的“公式单元格”进行解析与位移。
- 大范围填充可分批处理（避免 UI 阻塞）。
- 失败场景提示：无相邻列数据 / 只读会话 / 目标超范围。

## 测试策略（TDD）
- 单测：
  - 解析位移：A1 + 绝对/相对 + 整列/整行引用。
  - 多区域公式、混合大小写列名。
  - 解析失败回退行为。
- 交互逻辑：
  - 双击填充目标范围计算（左列优先、无数据时不触发）。
  - 仅向下扩展，不跨出 rowCount。

## 未来升级（仅记录，不实现）
- 后端解析/校验：
  - 多用户协同与冲突检测时，需要服务端解析/校验为权威来源。
  - API 写入/批量导入时，前端无法兜底解析，服务端应负责验证。
  - 审计/合规场景需要可重算与可追溯的公式处理链路。

# UI Polish Design (Loader Auto-Hide + Sheet Add Position + Formula Persistence)

## 背景
当前前端存在三处体验问题：
1) 底部调试条在加载完成后一直显示，占视觉注意力。
2) Sheet 的“+”按钮在最右侧，未贴近标签组。
3) 公式输入会被计算值覆盖，导致公式依赖/回显不稳定。

## 目标
- Loader 完成后 10 秒自动隐藏（仅针对“Loaded ... rows”完成提示）。
- Sheet “+”按钮紧贴最后一个标签（与 tab 在同一容器内）。
- 公式保存原始表达式，同时单元格显示计算值；公式栏显示原公式。

## 非目标（本轮不做）
- Pivot 交互与数据准备流程（后续专项计划处理）。

## 设计概览
### 1) Loader 自动隐藏
- 触发条件：debugInfo 显示“Loaded <table>: <n> rows”并且 loading=false。
- 行为：启动 10 秒计时器，超时后清空 debugInfo。
- 约束：只处理“加载完成提示”，不影响错误/警告提示。

### 2) Sheet “+”按钮位置
- 结构调整：将“+”按钮移动进 `.sheet-tabs` 容器，渲染在 tabs 列表末尾。
- 样式保持原视觉，不改变交互行为与可访问属性。

### 3) 公式回显与持久化（不改 DAG）
- 写入逻辑：保存原始公式字符串到缓存与后端（保持依赖链输入是公式）。
- 显示逻辑：对聚合/lookup 这类前端拦截的公式，渲染时使用“显示覆盖值”。
- 公式栏：继续显示原始公式文本（现有逻辑）。

## 数据流/状态说明（简化）
- debugInfo + timerRef：控制完成提示自动清空。
- sheetTabs：渲染时追加 add 按钮。
- formulaDisplayOverrides：记录单元格显示值覆盖，不影响底层公式存储。

## 风险与应对
- 覆盖显示值与缓存公式同步：需要保持 key 稳定（row,col），刷新后清理覆盖。
- Loader 定时器并发：新完成提示出现时需重置计时器。

## 验收标准
- 完成提示 10 秒后自动隐藏，其它提示不受影响。
- “+”按钮紧贴最后一个 tab。
- 输入公式后：单元格显示结果，公式栏仍显示原公式，且重新计算不被“数值覆盖”破坏。

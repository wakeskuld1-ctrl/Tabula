# 公式回显等待态（单元格内）设计文档

**日期：2026-03-14**

## 背景与问题
- 批量公式更新需要等待后端计算并刷新页面。
- 当前回显存在可感知延迟，用户缺乏明确的“正在计算”的反馈。

## 目标
- 在公式回显完成前，单元格内显示 `⏳ 计算中…`。
- 仅对涉及公式输入的单元格提示，不影响非公式编辑。
- 等后端刷新完成后自动恢复为真实计算值。

## 方案对比
### 方案A（推荐）：Pending Key 集合 + 渲染时覆盖显示
**优点**
- 不改后端，影响面可控
- 精准到单元格，不干扰其它数据
- 易于测试与回滚

**缺点**
- 需要新增 pending 状态管理

### 方案B：直接写缓存值为“计算中”
**优点**
- 实现简单

**缺点**
- 覆盖真实值，失败时可能不一致

### 方案C：公式文本拼接“⏳”
**优点**
- 实现简单

**缺点**
- 显示混乱且可能影响写回值

## 设计概述（方案A）
### 核心思路
- 新增纯函数 `collectFormulaPendingKeys`，计算本次编辑涉及的公式单元格 key。
- 组件内维护 `pendingFormulaKeys` 状态。
- `getCellContent` 渲染时若 key 在 pending 集合中，显示 `⏳ 计算中…`。
- 刷新完成或失败后清理 pending，避免“卡住”。

### 数据流
1. 批量编辑 / 单元格编辑 → 识别公式输入
2. 生成 pending keys 并加入状态
3. 提交后端
4. 刷新完成（或失败）后移除 pending keys

### 错误处理
- 批量提交失败：清理 pending，使用既有兜底刷新逻辑
- 刷新失败：记录日志并清理 pending，避免无限等待

## 测试策略（TDD）
- 新增最小脚本测试（node + assert），验证：
  - 公式输入生成正确的 pending key 集合
  - 非公式输入返回空集合

## 影响范围
- `GlideGrid.tsx`（渲染与批量编辑流程）
- 新增 `collectFormulaPendingKeys` 工具函数
- 新增轻量测试脚本

## 回滚策略
- 移除 pending 逻辑与工具函数即可回滚

# Fill Handle Advanced (Hitbox + Prefetch + Structured Ref) 设计文档

## 背景与目标
- 目标 1：细化填充把手命中范围，适配不同缩放与行高。
- 目标 2：双击填充在未缓存时自动补抓数据，避免提前停止。
- 目标 3：结构化引用支持按当前表列顺序进行“列偏移映射”。

## 非目标（本次不做）
- 不新增后端接口。
- 不引入后端公式解析/校验（仅记录未来升级项）。
- 不实现多人协同锁定机制。

## 现状摘要
- 已支持拖拽填充与双击填充入口，双击仅依赖缓存范围。
- 公式解析位移已支持整列/整行与相对/绝对引用。
- 结构化引用当前不位移。

## 方案选择（确认）
采用方案 B：
- 动态命中范围
- 未缓存补抓
- 结构化引用按当前表列顺序映射

---

## 设计方案

### 1) 填充把手命中范围
- 依据 `DataEditorRef.getBounds` 获取右下角单元格边界。
- hitbox 大小：
  - `handleSize = min(10px, cellHeight * 0.25)`
  - 增加 `±2px` 误差容忍区
- 只有双击命中 hitbox 才触发自动填充。

### 2) 未缓存自动补抓
- 当相邻列连续区间超出缓存时：
  - 计算缺失页号并调用 `fetchPage(page)`
  - 每拉一页继续向下扫描
  - 加上限制：最多补抓 3 页 / 或最多 1000 行（取先到）
- 若仍未遇到空行，以 `rowCount` 截断。

### 3) 结构化引用列映射
定义：结构化引用 = 按列名引用的公式，例如 `Table[Sales]`, `[@Sales]`。
- 解析结构化引用时只替换列名，保持原结构。
- 列名映射基准：当前表 `columns` 顺序。
- 映射规则：
  - 计算列名在 `columns` 中的索引
  - 叠加 dx 后取新列名
  - 越界或找不到列名则保持原样

---

## 数据流（双击填充）
1. 读取当前选区范围
2. 选择相邻列（左优先，右回退）
3. 若缓存不足，分页补抓
4. 得到 `destination` 后调用 `fillRange`

---

## 错误与边界处理
- 相邻列无数据 -> 不触发填充
- 未缓存补抓失败 -> 停止扩展，避免误写
- 结构化引用映射失败 -> 保持原样
- 超出 `rowCount` -> 截断范围

## 测试策略（TDD）
1. hitbox 命中：不同 cellHeight 下命中/未命中
2. 未缓存补抓：模拟缺页 -> 补抓 -> 扩展成功
3. 结构化引用映射：
   - `Table[Sales]` 右移 -> `Table[Profit]`
   - `[@Sales]` 右移 -> `[@Profit]`
   - 列名不存在/越界 -> 保持原样
